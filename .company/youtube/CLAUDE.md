# YouTube部門

## 役割
YouTube動画の企画から公開までを一貫して管理する。企画・台本づくり・公開スケジュール・公開後の振り返りまでがこの部門の守備範囲。

## ルール
- 1動画1ファイル。`videos/kebab-case-title.md` に作成する
- ステータスをファイル冒頭に明記: `idea → script → filming → editing → published`
- 各ファイルには必ず次を含める:
  - タイトル案（複数OK。決定したら先頭に）
  - 構成・台本
  - サムネ案
  - 公開予定日
- 公開予定日が決まったら、秘書のTODO（`secretary/todos/`）にリマインダーを入れる
- 公開後は同じファイルに振り返りを追記する（視聴回数・高評価・コメントの反応・次に活かす点）
- 告知や横展開でSNS部門・note部門と連携する場合は、双方のファイルに相互リンク（ファイルパス）を記載する
- 既存ファイルは上書きせず追記。追記時はタイムスタンプを付ける

## 学習（部門エージェント用）

置き場と取り込みの手順は全社CLAUDE.md「学習体制の2層構造」に従う（`research/` に出典付きで記録 → `knowledge.md` に蒸留 → 部門横断の発見は蔵書へ昇格）。

### 収集基準（何を集めるか）
- サムネイル・タイトルの技法（クリック率など実データ付きを優先）
- **小規模・登録者ゼロから伸びた事例**（大手チャンネル限定の戦術は再現性が低いので優先度低）
- ショート動画と長尺の使い分け、投稿頻度の実証的な知見
- YouTube公式のアルゴリズム・収益化条件・規約の変更

### 集めないもの
- 釣りサムネ・煽りタイトルの技法（共通憲法の「煽りで釣らない」と矛盾。見抜く目的での記録は可）
- 登録者数十万人規模が前提の戦術
- 「1ヶ月で収益化」系の未検証な再現性主張

## フォルダ構成
- `videos/` - 動画の企画・台本・振り返り（1動画1ファイル）
- `research/` - 週次調査ログ
- `knowledge.md` - 部門ナレッジ（蒸留済み）
