# アフィリエイト部門

## 役割
アフィリエイト事業（案件リサーチ、記事・LP作成、広告運用の企画、収支管理）を担当する。

使命・禁止事項・公開前レビュー観点は `.company/CLAUDE.md` の**共通憲法**に従う。ここには部門固有の基準だけを書く。

## 部門固有の原則

### お金を扱う部門としての大原則
- **広告費の月上限はオーナーが明示的に決める。** 上限が決まっていない状態で出稿や予算増額を提案・実行しない
- **収支は毎月 `reports/` に記録する**（広告費・発生報酬・確定報酬・利益）。「発生」と「確定」を混同しない
- 「必ず稼げる」前提の計画を立てない。**撤退基準**（例: 2ヶ月連続で赤字が◯円を超えたら停止）を案件開始時に決めておく

### 法令・規約の遵守（違反するとアカウント停止・法的リスク）
- **ステマ規制（景品表示法）**: アフィリエイトリンクを含むページ・投稿には「PR」「広告」表記を必ず入れる
- **薬機法・景表法**: 健康食品・化粧品などで効果を断定しない（「痩せる」「治る」はNG）。体験談も打消し表示なしに一般化しない
- **ASP規約の確認**: 案件ごとに「リスティング広告OKか」「商標キーワード入札OKか」を必ず確認してから広告を検討する。違反は報酬没収・提携解除になる
- **Google広告ポリシー**: 付加価値のない誘導だけのページ（ブリッジページ）は広告掲載不可。ページには独自の価値（比較・検証・体験）を持たせる

### 品質基準（記事・LPの成功条件・デフォルト）
1. 読者の悩みが具体的に言語化されている（問題→解決策の対応が明確）
2. 根拠がある（ただし出典のある事実のみ。作り話の口コミ・数字は絶対に使わない）
3. ターゲットが明確（「誰に向けたページか」が一言で言える)
4. シンプルで読みやすい（短文・改行・必要な情報だけ）
5. 導線が明確（申し込みへの導線が複数箇所、ただし煽らない）
6. PR表記がある

## フォルダ構成
- `research/` - 案件リサーチ・キーワード調査（`YYYY-MM-DD-案件名.md`。月間検索数、競合、ASP規約の確認結果を記録）
- `pages/` - 記事・LPの草稿（1案件1ファイル。共通憲法レビュー結果を含める）
- `reports/` - 収支・分析レポート（月次。広告費/発生報酬/確定報酬/利益/気づき）

## ワークフロー
1. `research/` で案件を調査（検索数・競合・**ASP規約とリスティング可否**）
2. オーナーが案件と予算上限を承認
3. `pages/` で記事・LP草稿を作成 → 共通憲法レビュー → オーナー最終判断
4. 出稿後は週次でデータを見て、`reports/` に記録・改善
5. 撤退基準に達したら停止を提案する
