# LP設計書: CHARGE OFFICE(オフィス出張整体)

> 本書はブリーフ(`charge-office-brief.md`)に基づくLP設計書。
> セクション順・見出し・コピーはこのまま実装に使える完成稿として書いている。
> 【要差し替え】マークの箇所のみ、クライアント確認後に実データへ差し替えること。

---

## 0. 設計方針

- **読者**: 従業員30名以上の企業の人事・総務担当者、経営層。BtoBなので「感情で掴み、数字で納得させる」バランスを取る
- **トーン**: 誠実で頼れる。煽りすぎない(Agitationも「危機感の可視化」に留め、恐怖訴求にしない。BtoBで過度な煽りは信頼を損なう)
- **1LP1ゴール**: 資料請求。LINE相談はハードルの低い副ゴール
- **CTAの原則**: ファーストビュー直下・Solution末尾・ページ末尾の計3箇所+追従ボタン。文言は「資料請求(無料)」で統一し迷わせない

## 1. カラー・デザイントークン

| 用途 | 色 | 備考 |
|------|-----|------|
| メイン(背景アクセント・見出し装飾) | 黄色 #FFC400 | 元気・回復のイメージ |
| サブ(CTA・強調・数字) | オレンジ #E8531F | 行動喚起はこの色に集約 |
| LINEボタンのみ | グリーン #06C755 | 公式規定色。他用途に使わない |
| テキスト | 濃グレー #333333 | 黒より柔らかく |
| 背景 | 白 / 薄クリーム #FFF9E8 | セクション交互で リズムを作る |

- フォント: 見出し=太ゴシック(Noto Sans JP Bold)、本文=Noto Sans JP Regular
- 角丸・影は控えめに。BtoBの信頼感を優先
- スマホファースト(人事担当者は移動中にスマホで見る想定)

## 2. ページ構成(PASONA準拠・全11セクション)

```
① ヘッダー(ロゴ+CTA小)
② ファーストビュー
③ 実績バー(数字3連)

```

## 3. セクション別コピー(完成稿)

### ② ファーストビュー

- **メインコピー**:
  **その肩こり、会社の経費で"ゼロ"にしませんか。**
- **サブコピー**:
  整体師がオフィスに出張。1人10分、たった0.5畳で始める新しい福利厚生 ——
  従業員の不調を、離職とコストの「見えない原因」のままにしない。
- **CTAボタン**: `資料請求はこちら(無料)`(オレンジ)/ `LINEで気軽に相談`(グリーン・小さめ)
- ビジュアル指示: オフィスの一角で専用チェアに座る社員+施術者。明るい黄色背景。【Canva素材】

### ③ 実績バー(数字3連・黄色帯)

| 施術実績 | 施術時間 | 必要スペース |
|---------|---------|-------------|
| **12万人** | 1人**10分**〜 | たった**0.5畳** |

### ④ Problem — 課題提起

- **見出し**: こんなお悩み、人事・総務のあなたに集まっていませんか?
- チェックリスト形式(✓アイコン):
  - 従業員の**離職が続き、採用コストが膨らんでいる**
  - **健康経営優良法人**を取りたいが、何から始めればいいか分からない
  - 福利厚生を充実させたいが、**場所も予算も手間もかけられない**
  - デスクワーク中心で、**肩こり・腰痛の不調を訴える社員が多い**
  - 健診結果は悪くないのに、**なんとなく生産性が上がらない**
- **締めの一文**: そのお悩み、バラバラの問題に見えて、根っこは一つかもしれません。——「従業員の身体の不調」です。

### ⑤ Agitation — 放置リスク

- **見出し**: 「病気じゃないから」と放置される不調が、組織を静かに蝕む
- 本文(3段落・簡潔に):
  1. 肩こりや腰痛は病気ではありません。だから病院にも行かず、誰にも報告されず、**会社の数字に表れないまま**生産性を下げ続けます。いわゆる「未病」の状態です。
  2. 不調を抱えた社員は、集中力が落ち、イライラし、やがて「この会社は自分を大事にしてくれない」と感じ始めます。**離職の理由は、退職届には書かれません。**
  3. 放置すれば、採用コスト・医療保険料・生産性低下という形で、**会社は静かに支払い続ける**ことになります。
- デザイン指示: このセクションのみ背景を薄グレーにしてトーンを落とす。ただし赤字・警告アイコン等の恐怖演出は使わない

### ⑥ Solution — サービス紹介

- **ブリッジコピー(黄色帯)**: その「見えないコスト」、オフィスに整体師を呼ぶだけで変わります。
- **見出し**: CHARGE OFFICE — 整体師がオフィスに来る、法人向け出張整体サブスク
- **3つの特徴(カード3枚)**:
  1. **驚きの高回転** — 1人10分から、1時間で最大5名。業務への影響を最小限に、より多くの社員をケア。
  2. **0.5畳あればいい** — 会議室の隅、休憩スペースの一角でOK。専用の折りたたみチェアで、施術ルームは不要。
  3. **選ばれた施術者だけ** — 独自研修に合格したスタッフのみが訪問。施術実績12万人の品質を、あなたのオフィスに。
- **導入効果(アイコン+短文の6グリッド)**: 離職率低下 / 採用コスト削減 / 健康経営優良法人の取得・継続支援 / 福利厚生の充実 / 医療保険料の負担軽減 / 社内コミュニケーション活性化【要確認: 最後の1項目はブリーフ外のため要承認】
- **CTA(2回目)**: `まずは資料でくわしく(無料)`

### ⑦ Solution — 導入までの流れ

- **見出し**: 導入はかんたん、最短4ステップ
- ステップ: ①資料請求・お問い合わせ → ②ヒアリング(規模・頻度・スペースのご相談) → ③契約・チェアのご購入 → ④施術スタート(整体師がオフィスへ)
- 【要確認】ステップ内容・所要日数はクライアント確認

### ⑧ Narrow down — 導入企業の声

- **見出し**: 導入企業の人事担当者さまの声
- 【要差し替え】実際の声をクライアントから入手するまで、以下はダミー構成(職種・規模のみの想定例として実装し、公開前に必ず差し替え):
  - IT企業・従業員50名・人事:「会議室の隅で始められたのが決め手。施術日は社内が明るくなります」
  - 士業事務所・従業員30名・総務:「健康経営優良法人の申請で、具体的な施策として説明できた」
  - メーカー・従業員120名・経営層:「福利厚生満足度調査で、導入後もっとも評価された施策になった」

### ⑨ Narrow down — 料金プラン

- **見出し**: シンプルな料金プラン
- **中央に大きく**: 1時間 **11,000円**(税込)
- 補足(表形式): 最低契約時間 3時間〜 / スタッフ出張費 1,100円/回 / 専用折りたたみチェア 契約時にご購入
- **換算コピー**: 1時間で最大5名 → **1人あたり約2,200円**。ランチ会1回分のコストで、社員の身体をメンテナンス。
- 【要確認】チェア価格はブリーフに記載なし。「詳細は資料にて」と逃がす設計にしてある(価格非開示はBtoB LPの資料請求フックとしても機能する)

### ⑩ Narrow down — FAQ(アコーディオン)

1. **Q. スペースが本当に0.5畳で足りますか?** — A. はい。専用の折りたたみチェア1台分のスペースがあれば施術可能です。会議室の一角や休憩スペースで導入いただいています。
2. **Q. 何名から利用できますか?** — A. 最低契約は3時間〜です。1時間で最大5名施術できるため、3時間で最大15名程度が目安です。
3. **Q. 服を脱ぐ必要はありますか?** — A. 着衣のまま受けられます。業務の合間に10分から、そのまま席に戻れます。【要確認】
4. **Q. 健康経営優良法人の申請に使えますか?** — A. はい。従業員の健康保持・増進施策の実績として、取得・継続をご支援します。
5. **Q. 支払い方法・契約期間は?** — A. 詳細は資料にてご案内しています。まずはお気軽にご請求ください。

### ⑪ Action — クロージング

- **見出し**: 従業員の身体を、会社の資産に。
- **本文**: 導入の検討には、まず現状の課題整理から。貴社の規模・ご予算にあわせたシミュレーション付きの資料を、無料でお送りします。
- **CTA大**: `資料請求はこちら(無料)`(オレンジ・最大サイズ)
- **サブCTA**: `LINEで気軽に相談する`(グリーン)
- **フォーム項目(最小構成)**: 会社名 / お名前 / メールアドレス / 従業員数(選択式: 〜29名・30〜99名・100名〜)/ お問い合わせ種別(資料請求・その他)
  - 項目は5つまで。BtoBフォームは1項目増えるごとに離脱が増えるため電話番号は必須にしない
- **追従ボタン(スマホ)**: 画面下部に `資料請求(無料)` を常時固定表示

## 4. Sonnet 5への実装指示

1. 単一HTMLファイル(CSS/JSインライン)で `.company/lp/charge-office/index.html` に作成(ブリーフ・設計書と同じLPフォルダ配下にまとめる)
2. レスポンシブ必須。スマホファーストで実装し、PCは2カラム展開
3. FAQはJSアコーディオン、追従CTAはスクロール連動(ファーストビュー通過後に表示)
4. フォームは送信先未定のため `form` タグ+ダミーaction で実装し、コメントで差し替え箇所を明示
5. 画像は全てプレースホルダー(黄色系の図形+ALTテキスト)にし、Canva素材の差し替え位置をコメントで明示 — 素材制作はオーナー担当
6. 【要差し替え】【要確認】マークはHTMLコメントとして残し、公開前チェックリストになるようにする
