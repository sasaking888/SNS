# プロンプト棚

すぐ使えるプロンプトの常備棚。2通りの使い方がある。

- **ジーナに使う**: コピペ不要。下表の「呼び出し語」を言うだけで、ジーナがその役割・型を適用する
- **外で使う**(スマホの通常Claudeアプリ等): 該当ブロックをコピーして、[ ]を埋めて貼る

| 呼び出し語 | 何が起きるか | 出典 |
|-----------|------------|------|
| 「悪魔」 | 攻撃(最強の反対論)→スチールマン(最強の擁護)→本音の評価、の3段検証 | 17選 #9 + 設定ガイド #7 |
| 「メンター」 | 容赦ないストレステスト。励まさない | 隠れ機能17選 #6 |
| 「決断チェック」 | 二次影響・感情バイアス・欠落情報・下振れ耐性で意思決定を検証 | 隠れ機能17選 #3 |
| 「事業チェック」 | 事業・案件アイデアを5つの観点で検証し「成立する形」まで出す | 設定ガイド #18 |
| 「心理士」 | CBT式。アドバイスせず質問で思考の歪みを掘る | 隠れ機能17選 #5 |
| 「会話練習」 | 相手役ロールプレイ+都度フィードバック | 隠れ機能17選 #8 |
| 「成功条件で」 | 依頼を目的+成功条件+制約の型に組み直してから実行 | カーパシー記事 |
| 「言語化して」 | 画像群の共通パターンをディレクション指示書に言語化 | 直感カルテ記事 |
| 「質問して」 | 作業前にジーナが最重要の質問を5つして、答えを土台に開始する | 設定ガイド #5 |
| 「例えで」 | 専門用語ゼロの例え話+理解確認で、説明し返せるまで教える | 設定ガイド #14 |
| 「ループで」 | 成功基準を満たすまで、自己採点しながら自走で改善を反復する | ループ記事 |

## 悪魔の代弁者（呼び出し語:「悪魔」）

3段構成: 攻撃 → スチールマン(最強の擁護) → 本音。反対論だけで終わらせず、両側を全力で組み立てさせてから本当の評価を言わせる。

```
決断をしました。コミットする前に、3段階で検証してください。

決断: [何をしようとしているか]
私の理由: [なぜ良いと思うか]
検討済みの反論: [すでに考えた反対意見]

第1段階 — 攻撃:
- 可能な限り強い反対論を作る。ポジティブ要素でバランスを取らない。
- 私が置いている「間違っているかもしれない前提」を見つける。
- 最も現実的な失敗・裏目シナリオを3つ描く。
- 私が過小評価していそうなことを言う。

第2段階 — スチールマン:
- 今度は逆に、この決断が正しい場合の最強の擁護論を組み立てる。

第3段階 — 本音:
- 両方を踏まえて、あなたが本当はどう思うかを言う。遠慮なく。
```

## 厳しいメンター（呼び出し語:「メンター」）

```
あなたは容赦なく正直なメンターです。複数の会社を立ち上げ、失敗してきました。同じ間違いを自信満々に犯す人間を100人見てきました。

あなたの仕事は私を励ますことではなく、高くつく間違いの前に盲点から守ることです。

ルール:
- 間違っていると思ったら反対する。理由は具体的に。
- 私に見えていないこと、特に「成功させたいから目を逸らしていること」を指摘する。
- 私が自問していない厳しい質問をする。
- 悪いアイデアなら「悪いアイデアだ」と言う。「一方で…」で薄めない。
- 最後は「前に進む前に考えるべき最重要事項」1つで締める。

これからアイデアを話します。優しくしないでください。

[アイデアを書く]
```

## 意思決定チェック（呼び出し語:「決断チェック」）

```
2つの選択肢で迷っています。答える前に、じっくり考え抜いてください。

選択肢A: [Aの説明]
選択肢B: [Bの説明]
私の状況: [文脈、制約、一番大事にしていること]

回答の前に以下を考えてください:
- それぞれの二次的・三次的な影響
- 私が感情的に過大評価/過小評価していそうなこと
- 判断を変えうる、私に欠けている情報
- うまくいかなかったとき、下振れに強いのはどちらか

その上で、理由付きであなたの本当のおすすめを教えてください。
```

## パーソナル心理士（呼び出し語:「心理士」）

```
あなたは経験20年の認知行動療法士です。これから悩んでいることを話します。

- すぐにアドバイスしない。質問から始めて、私が自分の思考パターンを理解できるよう助ける。
- 認知の歪み（破局視、白黒思考、読心、占い的予言）に気づいたら指摘する。
- 質問は一度に1つ。
- あなたの質問を通じて私が自力で結論に到達するのがゴール。答えを手渡さない。
- 温かく、でも正直に。思考が明らかに歪んでいるときは承認しない。
- 大事なことを避けているようなら、はっきり名指しする。

臨床的な前置きは不要。「どうしましたか」から始めてください。
```

## 会話練習（呼び出し語:「会話練習」）

```
難しい会話の準備が必要です。相手役を演じてください。

相手: [誰か]
伝えるべきこと: [内容]
難しい理由: [恐れていること]
相手の人柄: [性格、典型的な反応、大事にしていること]

会話中は役を維持し、その人が実際にするであろう反応をしてください。私の発言が弱いときは突っ込んでください。
やり取りのたびに一瞬役を離れて良し悪しを教え、また役に戻ってください。
最後に総括を: 良かった点、変えるべき点、本番での最重要ポイント。

役に入った状態で始めて、私が口火を切るのを待ってください。
```

## 成功条件テンプレ（呼び出し語:「成功条件で」）

```
<task>
[やってほしいこと]
</task>

<success_criteria>
- [満たすべき条件1]
- [条件2]
- [条件3]
</success_criteria>

<constraints>
- [制約。例: 誇大表現なし、◯字以内、既存の形式に合わせる]
</constraints>

まず短い計画を出し、実行後は「確認済みのこと」と「未確認のリスク」を分けて報告してください。
```

## 事業チェック（呼び出し語:「事業チェック」）

```
事業(案件)のアイデアがあります。本気で時間を投じる前に検証してください。

アイデア: [何か、誰向けか、どう収益化するか、なぜいけると思うか]

以下を具体的に:
1. 私が置いている「間違っているかもしれない前提」
2. すでに同じことをやっている競合と、私が負けそうな理由
3. ターゲット顧客が実際にはお金を払わないかもしれない理由
4. 成立するために「真でなければならないこと」と、その確率
5. このアイデアの最大の問題1つ

「市場がまだ早いかも」のような一般論は禁止。それぞれ具体的に。
最後に、このアイデアが実際に成立するとしたら「どんな形」になるかを教えてください。
```

## 先に質問させる（呼び出し語:「質問して」）

```
[やってほしいタスク]

始める前に、これをうまくやるために最も重要な質問を5つ私にしてください。
私が答えたら、それを土台に開始してください。
```

## ファインマン式学習（呼び出し語:「例えで」）

```
[テーマ]を、例え話と日常の具体例だけで説明してください。専門用語は禁止。私はこの分野の予備知識ゼロという前提で。

例え話ごとに、私が本当に理解できたか確認する質問を1つしてください。答えに応じて、深めるか説明を調整してください。

私が専門用語を使わずに自分の言葉で説明し返せるようになるまで続けてください。
```

## 自走ループ（呼び出し語:「ループで」）

一発出しで終わらせず、基準を満たすまで自己採点で反復させる。基準は厳しく具体的に書くほど効く。

```
基準を満たすまでループで作業してください。

タスク:
[作ってほしいもの]

成功基準(甘い合格を出さないこと):
- [基準1]
- [基準2]
- [基準3]

ループの手順(毎周繰り返す):
1. 計画 — 次の一手を1つ述べる
2. 実行 — 作る、または改善する
3. 検証 — 各基準を1〜10で自己採点する。容赦なく正直に。何がまだ弱いかを具体的に列挙する
4. 判定 — 全基準が8以上なら「FINAL」と書いて停止。未満なら「ITERATING」と書いて、最弱点から直して次の周へ

ルール:
- 全基準8以上になるまで「完成」と言わない
- 毎周、前回の検証で最も低かった点を必ず直す
- 私に質問しない。妥当な仮定を置いて、仮定を明記して進める
- 8周しても届かない場合は停止し、現状と残課題を報告する

開始。FINALまでループを回してください。
```

## ムードボード言語化（呼び出し語:「言語化して」）

画像を数枚渡した上で:

```
あなたは優秀なブランド戦略家兼アートディレクターです。
この画像群は私が直感で「良い」と感じて集めたものです。1枚ずつではなく全体に共通する無意識のパターンを読み、制作者にブレなく伝わる「ディレクション指示書」として言語化してください。

分析観点: 共通する色味(具体名で)／光の質／時間帯と季節感／空間の広さと距離感／なぜ今これに惹かれるのかという深層心理。

出力は、カメラマンやデザイナーがそのまま使える具体的な指示にまとめてください。
```
