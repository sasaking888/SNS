# 【全訳】Claudeの隠れた機能ガイド — ほとんどの人が知らない17の機能

> 出典: 英語のX記事(筆者不明)。ジーナによる日本語全訳。プロンプトはそのまま使える形に翻訳済み。
> 末尾に「分析メモ」あり(正確性の検証と、この会社への適用)。

---

Claudeは、ほとんどの人が思っているよりずっと多くのことができる。

このガイドはその全部をカバーする——各機能がどこにあるか、どうやってオンにするか、そしてすぐ使えるプロンプト。読んで、自分のワークフローに合うものを選んで、今日セットアップしてほしい。どれも数分で設定でき、その後は毎日効いてくる。

## 通常のClaudeの隠れ機能

### 1. Projects — あなたを本当に覚えているClaude

新しいチャットを開くたび、Claudeはゼロから始まる。あなたの名前も仕事も好みも知らない。ほとんどの人はこれを受け入れて、毎回自己紹介をやり直している。

Projectsはこれを解決する。プロジェクトを作り、資料をアップロードし、常設の指示を書いておく——Claudeはそれを永続的に保持する。来週開いても、前回の続きから始まる。

Projectsを使ったことがないなら、この記事の何よりも先にこれを直すべきだ。

**プロジェクト指示の例(Project Instructions欄に貼る)**

```
あなたは私のコンテンツリサーチアシスタントです。私は技術系読者向けにAIと暗号資産のニュースレターを運営しています。

読者は基礎を知っている前提で話してください。LLMとは何か、ブロックチェーンとは何かの説明は不要です。

私がトピックや記事を共有したら、あなたの仕事は:
1. 最も直感に反する・意外な切り口を3つ特定する
2. 私が見落としていそうな最近の出来事との接点を見つける
3. 要約ではなく「物語」として組み立てる方法を提案する

トーン: 直接的に。企業っぽい言い回しや埋め草は禁止。
形式: 短い段落で。指示しない限り箇条書きは使わない。
返答を「良い質問ですね」「もちろんです」で始めない。
```

### 2. Artifacts — チャットの中で動くアプリ

Claudeはテキストしか作れない、と思っている人が多い。間違いだ。

Artifactsは、Claudeがチャット内で「実際に動くもの」を作る機能。どこかにコピーするコード片ではなく、サイドパネルで動く生きた成果物。電卓、習慣トラッカー、ゲーム、グラフ付きダッシュボード。会話から離れずに、開いて、クリックして、使える。

SVG画像、インタラクティブなグラフ、Mermaid図——全部対応。無料プランで使える。ほとんどの人は試したことがない。

**試してみて**

```
習慣トラッカーを動くWebアプリとして作って。

毎日の習慣を5つ記録したい。

毎日チェックを付けられるようにして。

習慣ごとに7日間の連続達成カウンターを表示して。

1日サボったら連続記録はリセット。

デザイン: ダーク背景、クリーンでミニマルに。

チェックボックスはクリックして気持ちいいものに——完了時に小さなアニメーションを付けて。

ページを更新してもデータが消えないようにして。
```

### 3. Adaptive Thinking(拡張思考) — ワンランク上の推論

ほとんどのユーザーはこれをオンにしたことがない。拡張思考は、Claudeが答える前に問題を段階的に考え抜くモードで、その思考過程を全部見られる。

単純な質問には要らない。複雑な意思決定、戦略分析、パターンマッチではなく本当に考えてほしい場面——出力の差は大きい。

オンにして、いつもの質問をして、答えを比べてみてほしい。

**本物の意思決定に直面したときに使う**

```
2つの選択肢で迷っています。答える前に、じっくり考え抜いてください。

選択肢A: [Aの説明]
選択肢B: [Bの説明]

私の状況: [文脈、制約、一番大事にしていること]

回答の前に以下を考えてください:
- それぞれの選択肢の二次的・三次的な影響
- 私が感情的に過大評価/過小評価していそうなこと
- 判断を変えうる、私に欠けている情報
- 物事がうまくいかなかったとき、下振れに強いのはどちらか

その上で、理由付きであなたの本当のおすすめを教えてください。
```

### 4. Memory — あなたが誰かを知っているClaude

Memoryをオンにすると、Claudeは時間をかけてあなたのプロフィールを構築する。仕事、プロジェクト、好みのコミュニケーション、今取り組んでいること。

完全に新しいチャットを始めても、すでに文脈を知っている。二度と自己紹介しなくていい。

デフォルトはオフ。存在自体知らない人がほとんど。

```
以下を記憶して、二度と聞かないでください:

私の名前は[名前]。[会社/プロジェクト]で[役割]をしています。
今の主なフォーカスは[取り組んでいること]。
私の読者/顧客は[誰か]。

私が助けを求めたら、特に言わない限りこの文脈を前提にしてください。
好みのコミュニケーションスタイル: [率直/詳細/カジュアル/フォーマル]。
返答で嫌なもの: [例: 箇条書き、長い前置き、過剰な注意書き]。

これを全部いま記憶に保存してください。
```

## Claudeに役割を与える — 1つのプロンプトで全部変わる

Claudeは「AIアシスタント」である必要はない。具体的な役割を与えると全力でそれになりきる——質問の仕方、反論するポイント、見逃さないことが変わる。以下のプロンプトをコピーして、新しいチャットの最初に貼るだけ。

### 5. パーソナル心理士

多くの人はClaudeを「承認マシン」として使う。問題を話すと、「大変ですね」と5個の箇条書きアドバイスが返ってくる。

良いセラピーはそうじゃない。このプロンプトはClaudeをCBT(認知行動療法)セラピストに近づける——答えを与える代わりに質問し、承認する代わりに思考に挑戦してくる。

行ったり来たりし続けている決断、正体のつかめない不安、明確な外部視点が欲しい場面に。

```
あなたは経験20年の認知行動療法士です。これから悩んでいることを話します。

あなたのアプローチ:
- すぐにアドバイスしない。まず質問から始めて、私が自分の思考パターンを理解できるよう助けてください。
- 認知の歪み——破局視、白黒思考、読心、占い的予言——に耳を澄まし、気づいたら指摘してください。
- 質問は一度に1つ。圧倒しないでください。
- あなたの質問を通じて私が自力で結論に到達するのがゴール。答えを手渡さないでください。
- 温かく、でも正直に。私の思考が明らかに歪んでいるときは承認しないでください。
- 私が大事なことを避けているようなら、はっきり名指ししてください。

臨床的な自己紹介は不要。「どうしましたか」から始めてください。
```

### 6. 厳しいメンター

デフォルトのClaudeはあなたに同意する。アイデアに乗っかり、理屈を支持し、良い点を見つける。これはほぼ常に間違った振る舞いだ。

このプロンプトはそれを無効化する。承認をやめてストレステストを始める。弱い前提、抜けている考慮、プランが壊れる正確な場所を見つけてくる。

不快だ。だから効く。

```
あなたは容赦なく正直なメンターです。複数の会社を立ち上げ、失敗してきました。同じ間違いを自信満々に犯す人間を100人見てきました。

あなたの仕事は私を励ますことではありません——高くつく間違いを犯す前に、私自身の盲点から私を守ることです。

ルール:
- 私が間違っていると思ったら反対してください。理由は具体的に。
- 私に見えていないこと、特に「プランを成功させたいから目を逸らしていること」を指摘してください。
- 私が自問していない厳しい質問をしてください。
- 悪いアイデアなら「悪いアイデアだ」と言ってください。「とはいえ一方で…」で薄めないでください。
- 返答の最後は「前に進む前に考えるべき最重要事項」1つで締めてください。

これからアイデアを話します。優しくしないでください。
```

### 7. パーソナルトレーナー

一般的なフィットネス情報はどこにでもある。でもあなたのスケジュール、怪我、器具、本当の目標は考慮されていない。

Claudeに実際の数字を渡せば、実際のプランを組んでくれる。テンプレではない。あなたの状況に合わせて組まれ、経過報告に応じて調整されるプログラムだ。

```
あなたは専門のパーソナルトレーナー兼スポーツ栄養士です。完全なトレーニングプログラムを組んでください。

私の状況:
年齢: [年齢]
現在の体重/体組成: [詳細]
目標: [減量/筋肥大/持久力向上/健康維持]
使える器具: [ジム/自宅/ダンベルのみ 等]
週に使える日数: [数]
1回の時間: [分]
怪我や制限: [詳細 または「なし」]
現在のレベル: [初心者/中級/上級]

12週間のプログラムを作ってください。週ごとに種目、セット数、回数、休憩時間まで。なぜその構成なのかも説明してください——指示に従うだけでなく、理屈を理解したいので。開始後は毎週報告するので、様子に応じて調整してください。
```

### 8. 難しい会話の練習

ほとんどの人は難しい会話に準備なしで臨む。自分が言いたいことは分かっていても、相手が実際に何と返してくるかは分かっていない。

Claudeが相手を演じる。リアルに。相手が返すように返し、あなたの主張が弱ければ突っ込み、良い結果を「勝ち取らせる」。数回練習すれば、本番は楽になる。

```
難しい会話の準備が必要です。相手役を演じてください。練習したいので。

相手: [誰か——上司、クライアント、共同創業者など]
伝えるべきこと: [頼むこと/伝えること]
難しい理由: [恐れていること/起こりうるまずい展開]
相手の人柄: [性格、典型的な反応、大事にしていること]

会話中はずっと役を維持してください。私の望む反応ではなく、その人が実際にするであろう反応をしてください。私の発言が弱い・説得力がないときは突っ込んでください。

やり取りのたびに、一瞬だけ役を離れて「何が良くて何がダメだったか」を教えて、また役に戻ってください。最後に総括を: 良かった点、変えるべき点、本番で最も心に留めるべきこと。

役に入った状態で始めてください。私が口火を切るのを待ってください。
```

### 9. 悪魔の代弁者

もう心は決まっている。反論も考え尽くした。確信している。

その瞬間こそ、Claudeにその決断を攻撃させるべきタイミングだ。丁寧な懸念ではなく、全力の反対論。最も現実的な失敗シナリオ3つ。成功させたいがゆえに見えていないもの。

今の5分。コミットする前に。後ではなく。

```
決断をしました。コミットする前に、可能な限り最強の反対論を組み立ててください。

決断: [何をしようとしているか正確に]
私の理由: [なぜ良いと思うか]
検討済みの反論: [すでに考えた反対意見]

あなたの仕事:
- この決断への可能な限り強い反対論を作る。
- ポジティブ要素でバランスを取らない。私はもう信じているので、必要なのは反論だけ。
- 私が置いている「間違っているかもしれない前提」を見つける。
- 最も現実的な失敗・裏目シナリオを3つ描く。
- 私が過小評価していそうなことを言う。
- 「これが本当に悪いアイデアである場合、私は何を信じている必要があるか」を言う。

容赦なく。これが間違いなら、今知る必要があります。
```

## ほとんどの人が知らないプロダクト機能

### 10. Claude in Chrome — あなたの見ている画面が見えるClaude

多くの人はClaudeを別タブで使い、必要なものを手でコピペしている。それは苦行だ。

Claude in Chromeはブラウザ拡張で、開いているタブの中身をClaudeが見て、操作できる。ページを読み、リンクをクリックし、フォームを埋め、新しいURLに移動する。日本語でタスクを説明して、あとは任せる。

ChromeウェブストアでClaude for Chromeを検索→インストール→Claudeアカウントでサインイン。拡張アイコンをクリックするとサイドバーが開く。

**タスク例**

```
今開いている求人一覧ページで。

表示されている全求人から、職種名、会社名、給与レンジ(あれば)、上位3つの応募要件を抽出して。

給与の高い順に並べた比較表を作って。

複数ページあるなら、次のページへ進んで全件カバーするまで続けて。
```

### 11. Claude Cowork — デスクトップに住むClaude

Web版のClaudeはあなたのPCにアクセスできない。ファイルが見えない。全部手で貼り付けるしかない。

Coworkはデスクトップアプリで、Claudeがファイルシステムに直接アクセスできる。実際のファイルを読み、文書を編集し、新規作成し、フォルダを整理する——チャット欄に何もコピーせずに。

### 12. Scheduled Tasks — 寝ている間に働くClaude

多くの人はClaudeを「自分が起動するもの」として扱う。チャットを開き、依頼を打ち、出力を待ち、タブを閉じる。

Scheduled Tasksはそれを変える。タスクを一度設定すれば、指定の時刻・頻度でClaudeが自動実行する。毎朝。毎週月曜。毎時間。実行して、結果をフォルダに保存してくれる。

**スケジュールタスクの記述例**

```
平日の毎朝7:30に以下を実行:
1. 過去24時間のAIと暗号資産の主要ニュースを検索
2. 最重要の5本を選ぶ——意外性がある、直感に反する、作り手や投資家に実質的な影響があるものを優先
3. 各ニュースについて: 見出し、2文の要約、なぜ重要か
4. 結果を「brief-[日付].md」として/briefsフォルダに保存

トーンは率直で分析的に。無駄なし。3分で読める分量で。
```

### 13. CoworkのSkills — プラグインのように能力を追加

Skillsは、Coworkの中でClaudeに特定の能力を与える組み込み済みの指示セット。毎回説明する代わりに、一度インストールすればClaudeはそのタスクの扱い方を知っている——PowerPoint作成、PDF処理、特定のワークフロー実行など。

スマホのアプリのようなもの。素のスマホでも動くが、アプリを入れればもっと多くのことができる。

探し方: Cowork → Customize → Skills で導入済みを確認。追加は左サイドバーのBrowse pluginsから。

### 14. CLAUDE.md — 毎セッション自動で読まれるルール

CoworkとClaude Codeでは、プロジェクトフォルダにCLAUDE.mdを作れる。Claudeは毎セッションの開始時に、頼まれなくてもこれを読む。

コーディング規約。文体ルール。社内用語。ブランドボイス。一度書けば、そのプロジェクトの全セッションで永遠に守られる。

```
# プロジェクト: AIニュースレター

## このプロジェクトについて
作り手と投資家向けのAI・暗号資産の週刊ニュースレター。読者3.5万人。トーンは率直・分析的・時々不遜。

## 文章ルール
- 段落は短く。最大3文。
- 記事本文に箇条書きを使わない。散文のみ。
- ダッシュ(—)を使わない。ハイフンにするか文を組み替える。
- 形容詞より数字。「大幅な時間短縮」ではなく「3時間の短縮」と書く。
- 使用禁止語:「画期的」「ゲームチェンジャー」「活用する」(動詞として)。

## コンテンツルール
- 読者はLLMを知っている前提。基礎を説明しない。
- 最も意外な・直感に反することから書き始める。
- 全記事に具体的な「だから何?」を——読者が何をどう変えるべきか。

## ファイル構成
- 下書きは /drafts
- 公開済みは /published に日付プレフィックスで: YYYY-MM-DD-title.md
- リサーチメモは /research
```

### 15. Claude Code — ターミナルでコードを書き、テストし、直すAI

Claudeでコードが書けることを、まだ知らない人がいる。断片ではなく、本番レベルのコード、機能まるごと、複雑なリファクタリング。日本語で説明すればClaudeが書く。

Claude Codeはさらに一歩先だ。チャット窓ではなく、開発環境の中で直接動く。実際のコードベースを読み、新しいコードを書き、テストを実行し、エラーメッセージを読み、タスクが終わるまでループでバグを直す。

VS CodeとJetBrainsに統合できる。GitHub Actionsに組み込めば、何も触らずにPRの自動レビューや作成もできる。

### 16. Claude Design — ビジュアル作業のためのAI

このプロダクトの存在自体、ほとんど知られていない。Claude DesignはAnthropic Labsの別ツールで、ビジュアル作業用——プロダクト紹介の1枚もの、ピッチ資料、プロトタイプ、LPレイアウト。

必要なものを説明すれば、Claudeが作る。PPTX、Canva、PDF、HTMLに書き出せる。デザイナーでない人にとって、3時間のFigma作業が10分の会話に置き換わる。

アクセスは claude.ai/design へ直行するだけ。

### 17. Prompt Caching — API呼び出しのコスト90%削減(開発者向け)

Claude APIで開発している人向け。リクエストに大きな繰り返しコンテキスト(長いシステムプロンプト、参照文書、コードベース)が含まれるなら、毎回同じトークンの再処理にお金を払っていることになる。

Prompt Cachingはその内容をサーバー側に保存する。以降の呼び出しはキャッシュを再利用する。キャッシュされたトークンは最大90%のコスト削減。応答も目に見えて速くなる。

キャッシュしたいcontentブロックに `"cache_control": {"type": "ephemeral"}` を追加。キャッシュは5分間持続し、使うたびにタイマーがリセットされる。

## 分析メモ(ジーナ)

### 正確性の検証(当事者として)
- **全体として正確性は高い**。今日読んだ中で唯一、販促への誘導がない純粋なガイド記事。Projects/Artifacts/Memory/Chrome拡張/Cowork/Scheduled Tasks/Skills/CLAUDE.md/Claude Code/Claude Design/Prompt Cachingは全て実在し、説明もほぼ正しい
- 細部の誤り: コード例のモデルID「claude-opus-4-6」は不正確(実在はclaude-opus-4-8など)。#3は「Adaptive Thinking」と「Extended Thinking」の名称が混在(最新モデルではAdaptive Thinkingが既定化される流れで、記事の説明はやや旧仕様寄り)
- キャッシュの仕様(5分TTL、使用でリセット、最大90%減)は正しい

### この会社にとっての価値
